# P8-I18N — Locale + date formatting foundation

**Step ID:** `P8-I18N`  
**Flag:** `enable_locale_foundation` (default **off**)

Foundation for international users: locale picker, `Intl` date/time helpers, RTL for Arabic, and a small string table for settings + relative day labels.

## Modules

| File | Role |
|------|------|
| `public/js/flux-i18n.js` | `FluxI18n` — locale storage, formatters, settings card |
| `supabase/migrations/20260528500000_locale_foundation.sql` | Feature flag seed |

## Globals (when flag on)

| Global | Usage |
|--------|--------|
| `window.fluxFormatDate(d, style)` | `short` \| `long` \| `weekday` \| `monthDay` \| `monthDayYear` |
| `window.fluxFormatTime(d, { hour24 })` | Time string |
| `window.fluxFormatDateTime(d)` | Combined |
| `window.fluxRelativeDayLabel(iso)` | `Today` / `Tomorrow` translated |

## Locales

- `en-US` (default)
- `es-US`
- `fr-FR`
- `ar-SA` (sets `dir="rtl"` on `<html>`)

Stored in `flux_locale_v1` (localStorage / Flux namespaced save).

## Enable (dev)

```javascript
window.FLUX_EXPERIMENTS = { enable_locale_foundation: true };
await FluxFeatureFlags.load({ force: true });
FluxI18n.install();
```

Settings → **Appearance** → **Language & region**.

## Rollback

Disable flag; reload. Locale key remains in browser but formatters fall back to `en-US` behavior.

## Adoption (P8.11)

`window.fluxFmtDate` / `fmtFluxDate()` are always available (locale applies when `enable_locale_foundation` is on). High-traffic surfaces updated: task due chips, countdown, week strip, quick-add, exam conflict banner, mobile dashboard date, global search.
